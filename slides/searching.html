04 / 24
Konsep Dasar
Apa itu Searching?

Searching adalah proses menemukan lokasi data tertentu (key) di dalam sebuah koleksi data.

Contoh nyata:

  • Mencari kontak di HP.
  • Mencari NIM di database mahasiswa.
  • Mencari file di dalam folder.
05 / 24
Jenis Searching
Dua Algoritma Utama

1. Sequential Search

Mencari secara linear/berurutan.

2. Binary Search

Mencari dengan membagi dua wilayah pencarian.

Sequential Search
Konsep Sequential Search

Strategi paling sederhana: Cari dari awal sampai akhir sampai ketemu atau data habis.

"Seperti mencari buku satu per satu di rak dari ujung kiri ke kanan."
[10, 20, 30, 40]
Cari: 30

Pseudocode
Pseudocode Sequential Search
for i = 0 to n-1:
    if data[i] == key:
        return i  // Ditemukan!
return -1  // Tidak ditemukan

Sangat mudah diimplementasikan dengan for loop atau while loop.

Analisis
Analisis Sequential Search
📈Kompleksitas: O(n) - Waktu berbanding lurus dengan jumlah data.
✅Kelebihan: Data tidak perlu diurutkan (bebas).
❌Kekurangan: Lambat untuk data berjumlah besar.

Binary Search
Motivasi Binary Search

Bagaimana kalau kita tidak perlu mengecek satu per satu?

"Seperti mencari kata di kamus, kita tidak mulai dari halaman 1, tapi langsung buka di tengah."
11 / 24
Syarat Penting
⚠️ Syarat Mutlak

Binary search hanya bisa bekerja jika data sudah:

TERURUT (SORTED)

[10, 20, 30, 40, 50] // Bisa Binary Search
[40, 10, 50, 20, 30] // TIDAK BISA
12 / 24
Konsep
Cara Kerja Binary Search
1Tentukan nilai tengah (mid).
2Bandingkan data tengah dengan kunci.
3Jika kunci lebih kecil, buang setengah kanan.
4Jika kunci lebih besar, buang setengah kiri.
5Ulangi sampai ketemu atau rentang habis.

Pseudocode
Pseudocode Binary Search
while left <= right:
    mid = (left + right) / 2
    if data[mid] == key:
        return mid
    else if key < data[mid]:
        right = mid - 1
    else:
        left = mid + 1
return -1
15 / 24
Analisis
Analisis Binary Search
📈Kompleksitas: O(log n) - Sangat efisien untuk data besar.
✅Kelebihan: Sangat cepat.
❌Kekurangan: Data WAJIB terurut.
Contoh: Untuk 1 juta data, Sequential butuh max 1.000.000 cek, Binary cuma butuh max 20 cek!

Common Pitfalls
⚠️ Kesalahan Umum
❌Lupa melakukan sorting sebelum Binary Search.
❌Salah menentukan mid (pembulatan).
❌Kondisi loop (left <= right) yang salah menyebabkan infinite loop.

💡 Kesimpulan
🔹Searching adalah operasi fundamental.
🔹Sequential: Mudah, fleksibel, tapi lambat (O(n)).
🔹Binary: Cepat (O(log n)), tapi butuh data terurut.
