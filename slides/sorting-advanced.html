Skalabilitas
Masalah Skala Besar

“Bagaimana jika datanya berjumlah 1 Juta?”

Algoritma dasar (Bubble/Selection/Insertion) butuh waktu tahunan untuk mengurutkan 1 miliar data!
Ide
Ide Baru
🚫Jangan bandingkan satu-satu dari awal ke akhir terus menerus.
✂️Pecah masalahnya!

Lebih mudah mengurutkan 2 data daripada 100 data, bukan?

Strategi
Divide & Conquer
1️⃣Divide (Bagi): Pecah data menjadi bagian kecil.
2️⃣Conquer (Urutkan): Urutkan bagian terkecil tersebut.
3️⃣Combine (Gabung): Gabungkan kembali menjadi satu kesatuan yang utuh.
Merge Sort
Konsep Merge Sort
↔️Bagi array menjadi dua bagian terus menerus sampai tersisa 1 elemen.
📈Urutkan saat proses penggabungan.
⚡Sangat efisien untuk data masif.
// Snippet Ide Merge
void mergeSort(int arr[], int l, int r) {
  if (l < r) {
    int m = l + (r - l) / 2;
    mergeSort(arr, l, m);
    mergeSort(arr, m + 1, r);
    merge(arr, l, m, r);
  }
}

Visualisasi Merge Sort
Insight
Insight Merge Sort
🚀Keunggulan: Jauh lebih cepat untuk data besar dibanding Bubble/Selection.
📦Kelemahan: Membutuhkan memori tambahan untuk menyimpan bagian yang dipecah.
